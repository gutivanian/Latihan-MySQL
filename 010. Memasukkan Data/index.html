Memasukkan Data

Untuk memasukkan data baru ke dalam tabel di MySQL, Anda menggunakan perintah INSERT INTO. Perintah ini memungkinkan Anda untuk menambahkan satu baris data atau beberapa baris data sekaligus ke dalam tabel.

Memasukkan Satu Baris Data

Untuk memasukkan satu baris data ke dalam tabel kita bisa menggunakan perintah: 

        INSERT INTO nama_tabel (kolom1, kolom2, kolom3, ...)
        VALUES (nilai1, nilai2, nilai3, ...);

Contoh: 

        INSERT INTO users (username, email, registration_date)
        VALUES ('john', 'john@email.com', '2023-01-01');

Dalam contoh ini, data baru dengan username 'john', email 'john@email.com', dan registration_date '2023-01-01' akan dimasukkan ke dalam tabel users.

Memasukkan Beberapa Baris Data Sekaligus

Anda juga dapat memasukkan beberapa baris data sekaligus dengan satu perintah INSERT INTO. Contoh: 

        INSERT INTO users (username, email, registration_date)
        VALUES
        ('mary', 'mary@email.com', '2023-02-01'),
        ('alice', 'alice@email.com', '2023-03-01'),
        ('bob', 'bob@email.com', '2023-04-01');

Memasukkan Data Dengan Mengabaikan Nilai Default

Untuk memasukkan data ke dalam tabel MySQL dengan mengabaikan nilai default yang ditetapkan pada kolom-kolom tertentu, Anda perlu menentukan kolom-kolom mana yang ingin Anda isi dengan nilai spesifik dan mengabaikan nilai default yang mungkin ada.

Misalkan Anda memiliki tabel users dengan struktur seperti ini: 

        CREATE TABLE users (
            id INT AUTO_INCREMENT PRIMARY KEY,
            username VARCHAR(50) NOT NULL,
            email VARCHAR(100) NOT NULL,
            registration_date DATE DEFAULT CURRENT_DATE
        );

Di tabel ini, registration_date memiliki default value CURRENT_DATE, yang akan diisi secara otomatis saat data baru dimasukkan ke dalam tabel, kecuali Anda menyediakan nilai untuk kolom tersebut. Untuk memasukkan data dan mengabaikan nilai default registration_date, Anda perlu mencantumkan kolom-kolom mana yang akan Anda isi nilai secara eksplisit: 

        INSERT INTO users (username, email)
        VALUES ('john', 'john@email.com');

Dalam contoh ini, kita tidak mencantumkan kolom registration_date dalam pernyataan INSERT INTO. Oleh karena itu, nilai default CURRENT_DATE tidak akan diisikan ke dalam tabel.
