Tipe Data String

Tipe data string di MySQL digunakan untuk menyimpan data teks dalam berbagai format. Berikut adalah beberapa tipe data string yang umum digunakan di MySQL beserta penjelasan dan contoh-contohnya:

CHAR

  1. Menyimpan string dengan panjang tetap.
  2. Jika panjang string yang dimasukkan lebih pendek dari panjang yang ditentukan, sisa karakter akan diisi dengan spasi.

Contoh penggunaan: CHAR(10): Dapat menyimpan string dengan panjang tepat 10 karakter. Contoh: 'HELLO' akan disimpan sebagai 'HELLO ' (dengan lima spasi di akhir).

VARCHAR

  1. Menyimpan string dengan panjang bersifat variabel (bisa berubah).
  2. Lebih efisien dalam penggunaan ruang dibandingkan dengan CHAR untuk string yang bervariasi dalam panjang.

Contoh Penggunaan: VARCHAR(50): Dapat menyimpan string dengan panjang hingga 50 karakter. Contoh: 'HELLO' akan disimpan sebagai 'HELLO' (tanpa spasi tambahan).

TEXT

  1. Menyimpan string teks panjang yang tidak praktis untuk ditentukan panjangnya dengan presisi.
  2. Digunakan untuk kolom yang berisi teks panjang seperti deskripsi, artikel, atau catatan.

Contoh Penggunaan: TEXT: Dapat menyimpan string teks yang sangat panjang. Contoh: 'Ini adalah deskripsi panjang yang bisa mencakup banyak paragraf...'

BLOB

  1. Menyimpan data biner besar, seperti gambar atau file.
  2. Serupa dengan TEXT, tetapi dirancang untuk data biner.

ENUM

  1. Menyimpan string yang hanya bisa memiliki salah satu dari nilai yang ditentukan.
  2. Berguna untuk kolom yang memiliki pilihan tetap, seperti status atau kategori.

Contoh Penggunaan: ENUM('male', 'female', 'other'): Dapat menyimpan salah satu nilai dari 'male', 'female', atau 'other'.

SET

  1. Menyimpan string yang bisa berisi nol atau lebih dari nilai yang ditentukan.
  2. Berguna untuk kolom yang bisa memiliki beberapa pilihan dari sekumpulan nilai tetap.

Contoh Penggunaan: SET('swimming', 'running', 'cycling'): Dapat menyimpan nol atau lebih nilai dari 'swimming', 'running', 'cycling'. Contoh: 'swimming,running'
